Cameron's Coding Cheatsheet - Display and Positioning

Tag Table

Tag Description Example
position: static default position
position: relative Relative to the set default static position of the page On a website, when a user hovers over an information icon, a small tooltip appears next to it.
position: absolute Relative to closest positioned PARENT element. Can be pinned to any part of the page, but will move with the page. All other elements on the page will ignore the element and act like it is not present on the page. A pop-up notification or tooltip that appears inside a button or card but does not affect the layout of other elements. It is positioned relative to its nearest positioned ancestor.
position: fixed sets a box to a specific position on page, regardless of scrolling, A sticky navigation bar that stays at the top of the page while scrolling.
position: sticky removed from flow and remain when scrolling up to a certain pixel location a box that comes when you scroll down, but up to a certain pixel location it then sticks in place / A table header that stays visible at the top while scrolling down a long table.
z-index: 0 Depth of the element - wont affect static 0 is default, 1 is layer above
display: inline Box wraps tight around the element Navigation menu links displayed side by side in a horizontal row without breaking into new lines
display: block fill entire width of page by default. A button that takes up the full width of its container, stacking below other elements instead of sitting inline.
display: inline-block Inline-block elements can appear next to each other and we can specify their dimensions using the width and height properties. Navigation menu items (like buttons) that sit next to each other in a row but still allow width and height adjustments.
float: left/right; moves elements as far left or right. works for static or relative positioned elements. must specify width
clear: left, right, both, none; specifies how elements should behave when they bump into each other on the page. When multiple floated elements have different heights, elements can “bump” into each other. used after floating multiple elements clear: left - the left side of the element will not touch any other element within the same containing element.
